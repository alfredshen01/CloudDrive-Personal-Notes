### 1. 專案改版與舊資料清理

- 將專案從「日曆」改版為彩色轉黑白圖片轉換網站，保留登入功能；改版前先打 git tag `v0.1-before-overhaul` 保存舊狀態，確認重要檔案都已納入版本控制。
- 後端改用 Pillow 做灰階轉換（`/convert`），移除舊的 events API。
- 清理沒在用的死檔 `initdb/`。

**問題 1：沿用舊模板改版，擔心資料庫會變亂**
- 原因：舊專案的資料表結構同時被兩處描述——`initdb/01_create_tables.sql`（原始 SQL）與 alembic migration。其中 `initdb/` 根本沒被 docker-compose 掛載，是死檔，卻會讓人誤以為它有效；另外 `events` 表已不再使用卻仍留在資料庫。同一份資料有兩個來源，正是「會亂」的根源。
- 解決：確認 alembic 才是唯一真實來源後刪除 `initdb/`；`events` 閒置表暫時保留（日後要清就新增一支 `DROP TABLE` migration，而非回頭改舊檔）。

### 2. 圖片轉換管線（灰階、格式、方向）

完成上傳到下載的完整圖片管線，過程中解掉三個問題：中文檔名、照片方向、傳輸效能。

**問題 1：上傳顯示「連線失敗」，實際是中文檔名造成 500**
- 原因：這題繞了三層才找到真因——(1) 一開始以為是部署空檔的 502（容器重建時 API 短暫下線），但部署完仍失敗；(2) 接著懷疑 CORS／www 跨源，實測同源請求正常，排除；(3) 用瀏覽器 DevTools 看到真正是 HTTP 500。最終定位：使用者照片檔名含中文，後端把它放進 `Content-Disposition` HTTP header，而 HTTP header 只能用 latin-1 編碼，中文無法編碼，拋出 `UnicodeEncodeError`，成為未捕捉的 500。前端的 catch 把「回應不是 JSON」也一律報成「連線失敗」，才會誤導方向。
- 解決：`Content-Disposition` 改用 RFC 5987 寫法（ASCII 安全的 `filename` 加 UTF-8 的 `filename*`）；前端錯誤訊息改成帶 HTTP 狀態碼，避免再把不同錯誤混為一談。

**問題 2：直式照片轉出來變橫的**
- 原因：手機感光元件其實是橫的，直式照片靠 EXIF 的 Orientation 標記告訴看圖軟體「顯示時要轉 90 度」。Pillow 預設不會自動套用這個標記，而且轉成 PNG 後 EXIF 也被丟掉，所以顯示的是未旋轉的原始橫向像素。
- 解決：轉換前先用 `ImageOps.exif_transpose()` 依 EXIF 把像素實體轉正，再轉灰階。

**問題 3：以為轉換慢是伺服器 CPU 不好**
- 原因：實測純運算（解碼＋轉灰階＋編碼）只要約 0.1–1 秒，不是瓶頸。真正的時間花在資料傳輸：一次要上傳約 8MB 原圖加下載結果。而結果之所以大，是因為輸出的是無損 PNG，一張照片的灰階 PNG 往往跟原圖一樣大（6.5MB）。升級 CPU 幫助不大，該做的是縮小傳輸量。
- 解決：預覽改回傳有損 JPEG（小很多、顯示快）；下載讓使用者自己選無損 PNG 或有損 JPG。

### 3. 檔案儲存與分類系統

- 新增檔案儲存與分類系統：`files` 資料表、以 Docker volume 存檔、列表／下載／刪除 API、側邊資料夾 UI。

**問題 1：刪除測試帳號時被外鍵約束擋住**
- 原因：`events` 表的 `user_id` 還指向那些使用者，資料庫不允許刪掉「仍被引用」的列，否則會留下孤兒資料。這是舊模板殘留的 `events` 表造成的副作用。
- 解決：先刪子資料（`DELETE FROM events WHERE user_id IN (...)`）再刪使用者；新建的 `files` 表則直接設 `ondelete=CASCADE`，日後刪使用者會連帶清掉其檔案記錄。

### 4. 介面與部署設定調整

- 介面改為極簡灰階風；上傳介面精簡成矮橫條。
- nginx 加 `client_max_body_size 12M`，避免大圖被擋。

**問題 1：上傳區被撐到幾乎滿版**
- 原因：第一版只改了內距（padding），但 antd 的 Dragger 元件 `.ant-upload-drag` 預設帶 `height: 100%`，被它撐高；而且調整 flex 時下錯層級（真正包內容的是 `.ant-upload-drag-container` 不是 `.ant-upload-btn`），所以圖示和文字也沒排成橫排。
- 解決：對 `.ant-upload.ant-upload-drag` 設 `height: auto !important` 覆蓋預設，並把 flex 規則下到 `.ant-upload-drag-container`，做成矮橫條。
