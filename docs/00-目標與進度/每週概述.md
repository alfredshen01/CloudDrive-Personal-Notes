# 每週概述

> 每週 2–4 行的摘要，細節見各段連結的週日誌頁面。

## W1（05-05 – 05-11）：三層架構與日曆 CRUD

從零建立三層架構（React＋FastAPI＋PostgreSQL）：Git 與 Python／Node 環境、FastAPI 連 PostgreSQL、用 Vite 建 React 前端並串接後端，完成日曆應用的 CRUD 與月曆 UI。
→ [W1 週誌](../01-日誌/W1-0505~0511.md)

## W2（05-12 – 05-18）：CORS、UI 元件庫與 Docker 容器化

深化 CORS 機制，完成前端 CRUD 並套用 Ant Design；接著把應用容器化——Dockerfile、自訂網路、Volume、docker-compose 整合三服務，過程排除大量 Docker 環境問題（port／CORS、映像版本、磁碟空間、prune 誤刪網路等）。
→ [W2 週誌](../01-日誌/W2-0512~0518.md)

## W3（05-19 – 05-25）：雲端部署與使用者認證

把應用部署到 DigitalOcean Droplet（VPS），建立 GitHub Actions 自動部署；實作使用者認證（Alembic migration、JWT、bcrypt），排除部署與前後端一致性的連環問題（.env 缺失、帳號欄位改名、Hooks 順序等）。
→ [W3 週誌](../01-日誌/W3-0519~0525.md)

## W4（05-26 – 06-01）：網域、SSL 與 HTTPS

申請網域 `mit-project.me` 並上 SSL，從 HTTP 全面切換到 HTTPS（Let's Encrypt、nginx HTTPS server block 與 `/api/` proxy）；設定 cron 自動續期並驗證，補齊部署維運的最後一塊。
→ [W4 週誌](../01-日誌/W4-0526~0601.md)

## W5（06-02 – 06-08）：圖片轉換平台改版

將日曆專案改版為彩色轉黑白的圖片轉換平台：Pillow 灰階轉換、檔案儲存與分類系統、EXIF 轉正、有損／無損輸出選擇。核心除錯心得：錯誤訊息會誤導（「連線失敗」實為中文檔名引發的 500），要分層排查。
→ [W5 週誌](../01-日誌/W5-0602~0608.md)

## W6（06-16 – 06-22）：加入 CloudDrive、建立評測系統

接手 AI 助理模組。第一件事不是寫功能而是建尺：評測系統（evaluation harness）第一版——400 個測試案例、三種執行模式、由考官模型評分。確立「先有評測、再做功能」的開發順序。
→ [W6 週誌](../01-日誌/W6-0616~0622.md)

## W7（06-23 – 06-29）：環境建置、助理四功能、執行稽核

CloudDrive 本機完整跑起來；助理推進四項功能（模型選擇、自建 skill 進對話、結構化輸出修復、多組具名模型連線）；用資料庫三表交叉查核確認助理「宣稱刪除、實際未執行」，確立執行稽核（execution audit）方法：LLM 自述不能當證據、真做了必留資料庫稽核紀錄。
→ [W7 週誌](../01-日誌/W7-0623~0629.md)

## W8（06-30 – 07-06）：可靠性工程主體週

四條線並進：本機模型補上輸出格式約束（constrained decoding）、回覆改由實際執行結果組成（honest reporting）、單一步驟失敗不再中止整批計畫、兩輪防止重複生成修正。建立 temperature 掃描工具完成首次真模型量測，定位「破壞性規劃」為模型最不可靠的類別（通過率 0–40%），排定後續優先序。
→ [W8 週誌](../01-日誌/W8-0630~0706.md)

## W9（07-07 – 07-13）：關閉 thinking、記憶 v1、回想診斷閉環

以對照實驗（60 樣本）證明規劃器關閉模型 thinking 後通過率 60%→100%、延遲快約 10 倍，重複生成迴圈歸零；對話記憶 v1 落地並通過多輪評測（15/15）；回想能力 0/5 的誤診與翻案完成「診斷先於修復」的完整示範。週末三份筆記與正式報告成稿、PR #7 合併，研究線收攏。
→ [W9 週誌](../01-日誌/W9-0707~0713.md)

## W10（07-14 – 07-20，進行中）：V0.2 審查、筆記系統整併與建站

依 V0.2 需求書逐條審查 harness 模組：兩輪獨立審查取聯集共 9 項有效發現、0 誤報，並盤點五個驗證層的測試現況（真模型與端到端證據是主要缺口）；審查過程沉澱出證據分級（L0 檔案存在／L1 讀過內容／L2 執行觀察過）方法論。同週完成個人筆記系統整併並建成 GitHub Pages 網站（含站內搜尋修復），筆記逐檔改寫成面向一般讀者的寫法。
→ [W10 週誌](../01-日誌/W10-0714~0720.md)
