# 前端概念

> 蒸餾自五月日誌（React／Vite／antd／CORS 段落）與 6/24 CloudDrive 前端導讀、6/25 preview bug 分析（原始記錄見各日誌與 cloud_drive/local-docs）。

## React

### Hooks 規則

- Hooks（`useState`、`useEffect` 等）必須在 component 頂層呼叫，不能放在條件判斷或提前 `return` 之後。
- 原因：React 以「呼叫順序」而非名稱識別每個 hook，每次 render 必須呼叫相同數量與順序的 hooks；提前 return 讓某些 render 少呼叫 hook，內部索引對不上，狀態追蹤錯亂（實際症狀：登入／登出後頁面空白，重新整理才恢復）。

### 受控元件（Controlled Component）

- 第三方元件（如 react-big-calendar）沒有外部傳入狀態 prop 時自己維護狀態（uncontrolled）；一旦替換其內建 UI（自訂 toolbar），原本內部的狀態必須由外部 `useState` 接管，否則 callback 觸發了也沒有 state 接住、畫面不更新。

### 狀態的單一來源

- 同一份資料只存一個地方。曾因 token 同時存在 component state 與函式參數兩處、呼叫時漏傳參數，造成 `Bearer undefined` → 401 → 被誤判為登出。函式改為透過 closure 直接讀外層 state，任何地方呼叫都不會錯。

## CORS 心理模型

- 「來源」＝協定＋網域＋port，三者都相同才算同源（`:5173` 與 `:5174` 是不同 origin，precise match）。
- 同源政策是**瀏覽器**的安全機制；curl 等非瀏覽器工具不受限。
- 流程：瀏覽器送 Preflight（OPTIONS）帶 `Origin` → 後端回允許清單 → 瀏覽器決定放行與否。沒設 CORS 時請求可能有到後端，但瀏覽器把回應藏起來，JS 拿不到資料。
- 它防的是 CSRF 場景：惡意網站的 JS 在使用者瀏覽器執行，跨來源請求會自動夾帶目標網站 cookie；同源政策阻止冒充。攻擊者用 curl 直打拿不到 cookie，後端回 401。
- 前端 JS 要讀取自訂回應標頭（如 `Content-Disposition`、`X-File-Id`），後端要設 `expose_headers`，否則跨源時瀏覽器不給讀。
- 上線時 `allow_origins` 用環境變數注入真實網址，不用 `"*"`。

## Vite 與建置

- `VITE_` 開頭的環境變數才會被打包進前端；`import.meta.env.VITE_API_URL` 讀取；`.env.production` 供 `npm run build` 使用，開發時走 fallback。
- 前端 JS 在**使用者的瀏覽器**執行，`localhost` 指使用者的電腦不是伺服器——API 網址必須用環境變數依環境切換。
- Vite 模板自帶的 `index.css`（`color-scheme`、`#root` 樣式）會與第三方元件樣式衝突，新專案先清理。
- 瀏覽器以 URL 為 key 快取靜態資源，部署後看不到新版先強制重新整理（`Cmd/Ctrl+Shift+R`）。
- favicon：`<link rel="icon">` 指定；可同時給多格式（ICO 打包多尺寸、相容性最好；PNG 常用；SVG 無限縮放），瀏覽器自動選；檔案放 `public/`（Vite 靜態資源資料夾）才會被網站存取。

## Ant Design

- antd 5.x+ 使用 CSS-in-JS，class 名稱是執行期產生的 hash，靜態 CSS 選擇器蓋不到；主題客製走 `<ConfigProvider theme={{ token: { colorPrimary, ... } }}>`。
- antd 的 Calendar 只是日期選擇器；要顯示事件、切換檢視用 react-big-calendar，兩者可並用（antd 管按鈕／modal／表單）。

## CloudDrive 前端架構（6/24 導讀）

- **token 生命週期**：
  - access token 只存 Zustand 記憶體（無 localStorage）。
  - 每次頁面載入由 `AuthInitializer` 先打 `POST /auth/refresh`（HttpOnly refresh cookie），成功才 render router。
  - Axios interceptor 對 401 自動 refresh 重試；refresh 走無 interceptor 的獨立 client，避免無限迴圈。
- **狀態分工**：伺服器狀態用 TanStack Query（query key 工廠函式）；UI／上傳／auth 狀態用 Zustand。
- **「token 只在記憶體」的代價**：
  - `<a href>` 下載與 `<iframe src>` 預覽直連 URL 不帶 Authorization header → 401。
  - 正解是 Axios 抓 blob（會帶 token）再產 object URL——6/25 診斷出的 repo bug，後由隊友 PR #10 以此方式實作。
- **MSW 的盲點**：前端單元測試 mock 了後端沒有的路徑（`/preview/{id}/info`），把真實契約不一致蓋住——mock 保證的是「前端邏輯自洽」，不是「前後端契約正確」。
